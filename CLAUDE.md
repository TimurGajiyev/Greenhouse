## Что это за проект

GreenHouse — калькулятор подбора оборудования (equipment sizing) для гибридных и автономных (off-grid) энергосистем: солнечные панели (PV), накопители энергии (BESS), дизель-генераторы (DG). Ядро отвечает на два вопроса:

1. **simulate** — выдержит ли ЗАДАННАЯ конфигурация оборудования нагрузку и сколько она стоит за жизненный цикл (проверка вендорского предложения);
2. **optimize** — какие РАЗМЕРЫ оборудования оптимальны: минимум lifecycle cost (CAPEX + O&M + топливо) при заданной надёжности. Формальное имя задачи: reliability-constrained capacity sizing, решается через LP/MILP.

Референсная архитектура — NREL REopt (REopt.jl). Первый реальный кейс — завод в Йемене (off-grid, вендорское предложение из PDF).

## Инварианты фундамента (нарушать нельзя)

1. **Δt из данных.** Шаг времени всегда вычисляется из индекса ряда (`timestep_hours`). Энергия = мощность × Δt. Запрещено писать `range(8760)` и молча приравнивать kW к kWh. (Аналог `snapshot_weightings` в PyPSA и `time_steps_per_hour` в REopt.)
2. **Технологии опциональны.** Блоки PV/Battery/Diesel в `Scenario` — `Optional[...] = None`. Нет ключа в JSON — нет технологии в проекте. Математика ядра — суммы по имеющимся компонентам; пустое множество даёт ноль. Никаких спец-веток «если нет солнца» в физике.
3. **Непрерывный сайзинг.** Оптимизатор ищет непрерывные kW/kWh (как `dvSize` в REopt); перевод в штуки — постобработка `ceil(size / unit_size)` через поля `unit_kw` / `unit_kwh` схемы.
4. **Единицы и валюта.** Внутри ядра только kW / kWh / USD. Любая конвертация — на входе/выходе.
5. **Время tz-aware.** Все ряды приведены к часовому поясу площадки (Йемен: `Asia/Aden`). Naive-отметки в CSV трактуются как местное время площадки.
6. **Ворота валидации (gates).** pydantic на входных границах; энергобаланс проверяется assert-ом на каждом шаге: inflow == outflow ± 1e-6; каждый модуль покрыт pytest.
7. **Ядро — чистая функция** scenario → results (без скрытого состояния): так sensitivity-свипы почти бесплатны.
8. **Manifest на каждый расчёт:** run_id, git commit, hash входов, версия схемы, солвер и его статус (для оптимизатора — objective value, а где доступно — shadow prices).
9. **Слои не смешивать:** schema (вход) → profiles (ряды) → simulate/optimize (движок за интерфейсом) → results/KPI (выход). Движок v1 — PuLP; интерфейс должен позволять замену на PyPSA/oemof без переписывания слоёв вокруг.
10. **Стиль модулей:** одна публичная функция + приватные помощники с `_underscore` (как в profiles.py).

## REopt.jl — «оракул» (куда смотреть при сомнениях в формулировке)

Локальная копия ЕСТЬ: `reference/REopt.jl-master/` (в .gitignore; распакована из REopt_API-master.rar 2026-07-12). Веб-инструмент для сверки чисел: https://reopt.nrel.gov.

| Наша концепция | Где в REopt.jl |
|---|---|
| Размеры = переменные решения (`dvSize`, `dvStoragePower`, `dvStorageEnergy`) | `src/core/reopt.jl` |
| Цель: min lifecycle cost; штраф `dvUnservedLoad` (идея VOLL) | `src/core/reopt.jl` (выражение `Costs`, `ObjectivePenalties`) |
| Энергобаланс по часам | `src/constraints/load_balance.jl` |
| SOC-динамика: stored[t] = stored[t−1] + Δt·(η·charge − discharge/η) | `src/constraints/storage_constraints.jl` |
| Min/max размеры + лимит площади крыши/земли | `src/constraints/tech_constraints.jl` |
| Надёжность отдельно от сайзинга (цепь Маркова) | `src/outagesim/backup_reliability.jl` |
| Шаг времени как параметр | `time_steps_per_hour` |
| Опциональность технологии | `if haskey(d, "PV")` в `scenario.jl` |
| Off-grid доля покрытия нагрузки | `min_load_met_annual_fraction` |

Другие референсы: PyPSA (https://pypsa.org), oemof.solph, Offgridders (github.com/rl-institut, прецедент «приложение поверх фреймворка»), pvlib (https://pvlib-python.readthedocs.io), HOMER — концептуально: simulate → optimize → sensitivity.

## Верификационные якоря (Йемен; новый код обязан воспроизводить эту арифметику)

- Площадка: 15.28° N, 44.08° E · пояс Asia/Aden · крыша 8 500 м² · off-grid (сети нет).
- Нагрузка (synthetic v0): пик 700 kW в 10 рабочих часов, база 50 kW → 7 700 kWh/сутки.
- Вендорские размеры: PV 1500 kWp · BESS 3132 kWh · DG 1000 kW.
- Юниты оборудования: панель 0.58 kWp → 2587 шт ≈ 1500.46 kWp; шкаф батареи 261 kWh → 12 шт = 3132 kWh; PCS 125 kW → 12 шт = 1500 kW; генсет 1000 kW.
- Экономика: дизельная генерация ≈ $0.26/kWh; ориентиры CAPEX: PV ≈ $398/kW, BESS ≈ $196/kWh.
- Риски для стресс-сценариев: песчаные бури (PV ≈ 0 несколько суток; soiling ≈ 5%), жара (температурный derate выше 45 °C), перебои топлива (резерв 20 000 л).
