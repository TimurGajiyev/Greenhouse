# GreenHouse

Калькулятор подбора оборудования (equipment sizing) для гибридных и автономных
(off-grid) энергосистем: солнечные панели (PV), накопители энергии (BESS),
дизель-генераторы (DG). Референсная архитектура — NREL REopt.

Первый кейс — завод в Йемене (off-grid): проверка вендорского предложения
1500 kWp PV / 3132 kWh BESS / 1000 kW DG и поиск оптимальных размеров.

## Что сделано (хроника)

**Ядро (шаги 1–9 дорожной карты, все закрыты):**

| Слой | Модуль | Что делает |
|---|---|---|
| Контракт данных | `src/schema.py` | pydantic-валидация сценария (JSON); технологии опциональны; unit-размеры для перевода в штуки; 3 политики надёжности |
| Профиль нагрузки | `src/profiles.py` | синтетика или CSV (час/15 мин/сутки); Δt всегда из данных |
| Профиль солнца | `src/solar.py` | PVGIS TMY (fallback NASA POWER/кэш) → PVWatts; kW на 1 kWp |
| Симулятор | `src/simulate.py` | rule-based dispatch; assert энергобаланса на каждом шаге; окно отказа дизеля для стрессов |
| KPI | `src/kpi.py` | renewable fraction, LPSP, дизель kWh/$/литры |
| Экономика | `src/economics.py` | CRF, годовые издержки, NPC, LCOE, payback |
| Оптимизатор | `src/optimize.py` | LP (PuLP+HiGHS): dispatch (потоки) и sizing (размеры — переменные решения); manifest с паспортом солвера |
| Sensitivity | `src/sweep.py` | свипы цен, Pareto-фронт с «коленом», стресс-сценарии |

**Улучшения, перенесённые из reference-инструментов** (локальные копии
REopt.jl и Calliope лежат в `reference/`): циклическое хранилище
(годовое кольцо SOC, дефолт сайзера), саморазряд батареи, запрет
переполнения зарядом (REopt 4h), LP-snapshot тесты (эталонные .lp-файлы,
паттерн Calliope), кросс-солверная сверка HiGHS↔CBC, анти-вырожденный
микроштраф размеров.

**Интерфейсы:** ноутбук `GreenHouse_analysis.ipynb` (весь разбор с
глоссарием терминов после каждого блока) и Streamlit-приложение `app.py`
(8 вкладок, ползунки-overrides по паттерну Calliope, словарь терминов в
сайдбаре, пояснение в конце каждой вкладки).

**Качество:** 94 pytest-теста; покрытие src/ — 93% (economics и kpi —
100%); экономика сверена вручную и против трёх независимых источников
формул (REopt pwf, Calliope depreciation_rate, PVGIS PVcalc).

**Внешние проверки (battle-тесты):**

| Кейс | Тип проверки | Итог |
|---|---|---|
| Тихоокеанский остров (Тонга), клиника, профиль NREL CRB | наш оптимум против ожиданий HOMER Pro | LCOE $0.270/kWh — внутри диапазона 0.25–0.32; расхождение состава (батарея −89%) объяснено плоской ценой топлива — задокументированное упрощение v1 |
| Рудник DeGrussa (Австралия), факты ARENA | симуляция as-built против измеренных итогов | CO₂-экономия −4% от факта; доля солнца −22% — целиком объясняется трекерами (у них) против фиксированного наклона (у нас) |
| REopt API (NREL) | одинаковые входы в оба инструмента | вход подготовлен (`scripts/validate_reopt_api.py`), но developer.nrel.gov geo-блокирован вне США; обход: `--export-inputs` → запуск `scripts/reopt_remote_submit.py` через VPN/Colab → `--compare` |

Вывод проверок: модель системно **консервативна** (занижает солнце, а не
завышает) — правильная сторона ошибки для проверки вендорских обещаний.

## Как запустить (Windows)

```powershell
python -m venv .venv                  # создать окружение (один раз)
.venv\Scripts\activate                # активировать (в каждом новом терминале)
pip install -r requirements.txt      # восстановить зависимости
pytest                               # прогнать все тесты (~1 мин)
```

### Готовые команды (все — из корня проекта)

| Команда | Что покажет | Время |
|---|---|---|
| `streamlit run app.py` | **Интерактивный интерфейс**: ползунки цен/нагрузки/надёжности → пересчёт оптимума, спецификация в штуках, схема системы, сравнение сценариев, сверка с REopt/HOMER | старт ~10 c, пересчёт ~5 c |
| ноутбук `GreenHouse_analysis.ipynb` | весь разбор по шагам с пояснениями и глоссарием (ядро VS Code: *Python (GreenHouse .venv)*) | Run All ~4 мин |
| `python scripts/run_full_report.py` | **ВСЁ**: проверка вендора → оптимальные размеры (kW и штуки) → sensitivity; графики и манифесты в `results/` | ~3 мин |
| `python scripts/show_solar.py` | солнечный профиль: типовые сутки и месяцы | ~5 c |
| `python scripts/show_results.py` | вендорская конфигурация: диспетчеризация, CAPEX, сравнение с чистым дизелем | ~10 c |
| `python scripts/show_sweeps.py` | sensitivity: tornado, Pareto с коленом, стрессы | ~2 мин |
| `python scripts/battle_test_pacific.py` | battle-тест №1: остров Тонга против ожиданий HOMER (сам качает профиль NREL и погоду) | ~1 мин |
| `python scripts/battle_test_degrussa.py` | battle-тест №2: рудник DeGrussa против фактов ARENA | ~1 мин |
| `python scripts/validate_reopt_api.py --export-inputs f.json` | подготовка входов для сверки с NREL REopt (см. geo-блок выше) | ~10 c |
| `python scripts/regen_lp_baselines.py` | перегенерация эталонных .lp (только при ОСОЗНАННОМ изменении математики) | ~5 c |

Входные сценарии: `scenarios/yemen_vendor.json` (проверка вендора, размеры
зафиксированы), `scenarios/yemen_sizing.json` (поиск оптимума, коридоры
открыты), `scenarios/pacific_island_test.json` (battle-тест Тонга).
Свой проект = свой JSON по тому же контракту.

Выходы в `results/`: `run_*.parquet` (почасовые таблицы прогонов),
`run_*_manifest.json` (паспорт каждого прогона: git commit, hash входов,
солвер), `*.png` (графики), `sweeps/` (сводные таблицы sensitivity).

## Упрощения v1 (сознательные)

- Топливо дизеля — плоские $/kWh (без кривой частичной загрузки и
  минимальной загрузки генсета); цена упрощения измерена battle-тестом
  Тонги — кандидат №1 в v1.1.
- PV — фиксированный наклон (без трекеров); цена измерена battle-тестом
  DeGrussa (~ −20% выработки против одноосных трекеров) — кандидат №2.
- Параметры модуля PV (КПД инвертора 0.96, темп. коэффициент −0.47%/°C,
  DC/AC 1.2) — константы `src/solar.py`, а не поля сценария.
- CRF по сроку жизни технологии: замены и остаточная стоимость не
  моделируются явно.
- LP-сайзер обладает «идеальным предвидением» погоды и нагрузки —
  реальный контроллер работает чуть хуже (разрыв измерен: ~0.2% LPSP;
  вкладка «Rule vs LP» в приложении).
- Нет: деградации батарей, сетевого подключения и тарифов, экспорта,
  CHP/тепла, мультигодовых сценариев, эмиссий в ядре (CO₂ — оценка
  в отчётах по фактору 0.72 кг/kWh дизеля).
- Равномерный шаг времени (15 мин / час / сутки поддержаны; месячные
  веса — нет).
